  <name>Task 1: Update i18n config for Spanish-first detection</name>
  <files>src/i18n/config.ts</files>
  <action>
Modify the i18n configuration to implement Spanish-first browser detection:

1. Add `supportedLngs: ['en', 'es']` to constrain detection to available languages only

2. Replace `fallbackLng: 'en'` with a custom fallback function that implements Spanish-first logic:
   - If detected language starts with 'en' (en, en-US, en-GB, etc.), return ['en']
   - For all other languages (es, fr, de, zh, or no detection), return ['es']

3. Keep existing detection config:
   - order: ['localStorage', 'navigator'] (user choice first, then browser)
   - caches: ['localStorage'] (persist user choice)
   - lookupLocalStorage: 'i18nextLng' (standard key)

4. Keep the window.i18n exposure for development testing

Pattern from research:
```typescript
fallbackLng: (code) => {
  if (code && code.toLowerCase().startsWith('en')) {
    return ['en'];
  }
  return ['es'];
},
```
  </action>
  <verify>
Run `npm run dev` and test in browser console:
- `localStorage.removeItem('i18nextLng'); location.reload()` then check `i18n.language`
- With English browser: should show 'en'
- With Spanish/other browser: should show 'es'
- After manual switch via UI, preference persists across reload
  </verify>

- Spanish-first fallback: English browsers get English, everything else gets Spanish
